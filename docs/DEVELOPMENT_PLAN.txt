================================================================================
  FINANCIAL ANALYSIS ASSISTANT — DEVELOPMENT PHASES PLAN
  For 2 Beginner Developers | Complete Build-to-Deployment Guide
================================================================================

This plan covers every step from your first line of code to a live deployment.
The project is split into two parallel tracks so both developers can work
simultaneously without blocking each other.

  DEV A  = Backend + ML Model developer
  DEV B  = Frontend developer

After the Pre-Phase (a short setup done together), both devs split and work
independently until the Integration Phases at the end.

================================================================================
  HOW TO READ THIS PLAN
================================================================================

  [BOTH]  = Both developers work on this together
  [DEV A] = Backend/ML developer does this
  [DEV B] = Frontend developer does this

  Each phase has:
  - What you will build
  - Step-by-step tasks (in order)
  - How to check it is working before moving on

  TIP: Always finish the checklist items in order. Do not skip ahead.

================================================================================
  PRE-PHASE: PROJECT SETUP & SHARED CONTRACTS
  Estimated time: 1 day | [BOTH]
================================================================================

  This is the ONLY phase where both developers must work together.
  After this, you split into independent tracks.

  GOAL:
  Agree on what the API will look like so Dev B can build the frontend
  against mock (fake) responses without waiting for Dev A to finish the
  real backend.

  STEPS:
  -------

  1. Set up your machine
     - Install Python 3.11+
     - Install Node.js v20+
     - Install PostgreSQL 15+
     - Install Git
     - Install VS Code + extensions: Python, ESLint, Prettier

  2. Clone the repository on both machines
     - git clone <your-repo-url>
     - cd prototype

  3. Set up Python virtual environment (Dev A does, Dev B watches)
     - python -m venv .venv
     - .venv\Scripts\activate    (Windows)
     - pip install -r backend/requirements.txt

  4. Set up the frontend (Dev B does, Dev A watches)
     - cd frontend
     - npm install

  5. Together, write the API contracts document
     Create: docs/api-contracts.md
     For each of the 13 analysis features, write down:
       - The URL (e.g. POST /api/v1/profit)
       - What JSON the frontend will SEND to the backend
       - What JSON the backend will SEND BACK to the frontend

     Example for Profit Analysis:
       URL: POST /api/v1/profit
       Send:   { "branch": "Bangalore", "period": "2026-Q2" }
       Receive: { "analysis": "..text..", "sources": { "period": "2026-Q2" } }

     Do this for all 13 endpoints:
       /financial-summary, /profit, /loss, /branch-analysis,
       /revenue, /expense, /balance-sheet, /cash-flow,
       /health-score, /trend, /risk, /report, /recommendations

  6. Together, create mock (fake) response JSON files
     Create one file per feature in: docs/mock-responses/
       docs/mock-responses/profit.json
       docs/mock-responses/loss.json
       ... (one per endpoint)

     Each file contains a realistic fake response the frontend can use.

  7. Set up the database locally (Dev A)
     - Create a PostgreSQL database called "financial_db"
     - Create a .env file in backend/ with:
         POSTGRES_SERVER=localhost
         POSTGRES_USER=postgres
         POSTGRES_PASSWORD=your_password
         POSTGRES_DB=financial_db
         SECRET_KEY=any_random_string_here

  CHECK: Both developers can run their respective dev servers:
    Dev A: cd backend && uvicorn app.main:app --reload → opens at localhost:8000
    Dev B: cd frontend && npm run dev → opens at localhost:3000

================================================================================
  PHASE A1: DATABASE FOUNDATION
  Estimated time: 3-4 days | [DEV A]
================================================================================

  GOAL:
  Create all the database tables the application needs and fill them with
  sample (test) data.

  WHAT YOU WILL LEARN:
  - How SQLAlchemy ORM works
  - How Alembic runs database migrations
  - How to structure database models

  STEPS:
  -------

  1. Create the database base file
     File: backend/app/db/base.py
     This file just creates the "Base" class that all models inherit from.

  2. Create the async database session
     File: backend/app/db/session.py
     This creates the connection to PostgreSQL using asyncpg.
     Use the DATABASE_URL from your .env file.

  3. Create all ORM model files (one file per table):
     backend/app/db/models/branch.py
       - branch_id, branch_name, region

     backend/app/db/models/financial_periods.py
       - period_id, branch_id, period_type, start_date, end_date

     backend/app/db/models/income_statement.py
       - period_id, revenue, cogs, operating_expenses, net_profit

     backend/app/db/models/balance_sheet.py
       - period_id, total_assets, total_liabilities, total_equity,
         current_assets, current_liabilities

     backend/app/db/models/cash_flow.py
       - period_id, operating_cf, investing_cf, financing_cf

     backend/app/db/models/expense_breakdown.py
       - period_id, category, amount

     backend/app/db/models/audit_log.py
       - log_id, user_id, query_text, retrieved_record_ids,
         response_summary, timestamp

  4. Set up Alembic (database migration tool)
     - cd backend
     - alembic init alembic
     - Edit alembic/env.py to import your models
     - alembic revision --autogenerate -m "initial tables"
     - alembic upgrade head

  5. Write a seed script to insert sample data
     File: backend/app/db/seed.py
     Add 2-3 branches, 4 quarters of financial data for each branch.
     Run: python app/db/seed.py

  CHECK: Open pgAdmin or a terminal and run:
    SELECT * FROM branches;
    SELECT * FROM income_statement;
    Both should show data. If yes, Phase A1 is done.

================================================================================
  PHASE A3: INTENT DETECTION
  Estimated time: 2-3 days | [DEV A]
================================================================================

  GOAL:
  Build the module that reads the user's natural language question and
  decides which of the 13 analysis types they want, plus extracts
  details like branch name and date range.

  WHAT YOU WILL LEARN:
  - Basic text classification (rule-based, no ML required yet)
  - How to extract entities from text (branch names, dates)

  STEPS:
  -------

  1. Build the intent detector
     File: backend/app/intent/detector.py

     Start with simple keyword matching:
       - If "profit" or "earning" in query → intent = "profit_analysis"
       - If "loss" or "losing" in query → intent = "loss_analysis"
       - If "balance sheet" or "assets" in query → intent = "balance_sheet"
       - If "cash flow" in query → intent = "cash_flow"
       ... and so on for all 13 types

     The function signature should be:
       def detect_intent(query: str) -> tuple[str, dict]
       Returns: ("profit_analysis", {"branch": "Bangalore", "period": "Q2"})

  2. Build the entity extractor
     File: backend/app/intent/entities.py

     Extract:
       - Branch name: look for known branch names in the text
         (Bangalore, Mysore, etc. from your seed data)
       - Date range: look for patterns like "Q1 2026", "January 2026",
         "last quarter", "this year"

  3. Write tests for both files
     File: backend/tests/intent/test_detector.py
     Test at least 10 different sample queries.

  CHECK: Run tests:
    pytest tests/intent/ -v
    All tests should pass. No server needed.

================================================================================
  PHASE A5: ML MODEL — TOKENIZER & TRANSFORMER
  Estimated time: 5-7 days | [DEV A]
  (Run this in parallel with A1, A2, A3 — they are independent)
================================================================================

  GOAL:
  Build the custom AI model from scratch. This runs independently of the
  backend — it is purely Python + PyTorch.

  WHAT YOU WILL LEARN:
  - How SentencePiece tokenization works
  - How a Transformer neural network is structured
  - How to train a language model

  STEPS:
  -------

  STEP 5A: Build the tokenizer
  1. Collect a small sample of financial text (public sources):
     - Financial news articles (plain text)
     - Public earnings report summaries
     - Accounting/finance Wikipedia pages
     Save all text to: model/tokenizer/corpus.txt

  2. Train the SentencePiece tokenizer
     File: model/tokenizer/train_tokenizer.py
       import sentencepiece as spm
       spm.SentencePieceTrainer.train(
           input='tokenizer/corpus.txt',
           model_prefix='tokenizer/financial',
           vocab_size=16000,
           model_type='bpe'
       )
     Run: python model/tokenizer/train_tokenizer.py
     Output: model/tokenizer/financial.model

  3. Test the tokenizer
     Load financial.model and tokenize sample sentences.
     Check that financial words like "revenue", "EBITDA", "amortization"
     get reasonable tokenizations.

  STEP 5B: Build the Transformer architecture
  4. Define model configuration
     File: model/architecture/config.py (already created)
     Keep it simple to start:
       - n_layer = 6
       - n_head = 6
       - n_embd = 384
       - context_window = 512
       - vocab_size = 16000

  5. Build the transformer blocks
     File: model/architecture/transformer.py
     Build these classes in order:
       a. CausalSelfAttention - the attention mechanism
       b. MLP - the feed-forward layer
       c. Block - one transformer layer (attention + MLP)
       d. FinancialLM - the full model (embedding + N blocks + output head)

     TIP: Follow a simple GPT-style tutorial online and adapt it.
     The key property: it is decoder-only (causal masking).

  6. Test the forward pass
     Run in Python:
       from architecture.transformer import FinancialLM
       from architecture.config import FinancialModelConfig
       import torch
       cfg = FinancialModelConfig()
       model = FinancialLM(cfg)
       x = torch.randint(0, 16000, (1, 128))  # fake token input
       out = model(x)
       print(out.shape)  # should print: torch.Size([1, 128, 16000])
     If this works, the architecture is correct.

  STEP 5C: Training (run on GPU if available, CPU is slow but works)
  7. Build the dataset loader
     File: model/training/dataset.py
     Loads text from your corpus, tokenizes it, returns sliding windows
     of 512 tokens as training examples.

  8. Write the training loop
     File: model/training/train.py
     - Load dataset
     - Create model
     - AdamW optimizer, cosine learning rate schedule
     - Train for next-token prediction (CrossEntropyLoss)
     - Save checkpoint every 1000 steps to model/training/checkpoints/

  9. Evaluate the model
     File: model/eval/evaluate.py
     Calculate perplexity on a small held-out set.
     Generate a few sample completions to check quality.

  STEP 5D: Export for CPU inference
  10. Quantize the model to INT8
      File: model/export/quantize.py
      Use PyTorch's built-in quantization:
        torch.quantization.quantize_dynamic(model, ...)
      Check that the quantized model is under 1GB.

  11. Export to ONNX format
      File: model/export/export_onnx.py
      torch.onnx.export(model, ...)
      Test that ONNX Runtime can load and run it on CPU.

  CHECK:
    - Tokenizer: python train_tokenizer.py → financial.model file created
    - Architecture: forward pass test above passes without errors
    - Training: loss decreases over time (watch the printed loss values)
    - Export: onnxruntime can load and run the exported model

================================================================================
  PHASE A6: WIRE IN THE LLM
  Estimated time: 2-3 days | [DEV A]
  (After A4 and A5 are both complete)
================================================================================

  GOAL:
  Connect the trained and exported ONNX model to the FastAPI backend
  so the API returns real AI-generated text instead of mock responses.

  STEPS:
  -------

  1. Build the inference client
     File: backend/app/inference/client.py
       - Load the ONNX model once at server startup (not on every request)
       - Provide a generate(prompt: str, max_tokens: int) -> str function
       - Run on CPU using ONNX Runtime

  2. Build the prompt formatter
     File: backend/app/inference/prompt_formatter.py
       - Takes: context text (from context builder) + user query
       - Returns: a single formatted string for the model
       Example format:
         "Financial Data:
          Branch: Bangalore | Revenue: 4200000 | ...
          Question: Why is this branch making a loss?
          Analysis:"

  3. Update every feature's service.py
     Replace: return load_mock("profit.json")
     With:
       context = context_builder.build(data)
       prompt = prompt_formatter.format(context, request.query)
       analysis = inference_client.generate(prompt)
       return ProfitResponse(analysis=analysis, sources=sources)

  CHECK:
    curl -X POST http://localhost:8000/api/v1/profit \
      -H "Authorization: Bearer <your_token>" \
      -d '{"branch": "Bangalore", "period": "2026-Q2"}'
    Response should contain real AI-generated text.

================================================================================
  PHASE D2: SERVER DEPLOYMENT
  Estimated time: 2-3 days | [BOTH]
================================================================================

  GOAL:
  Deploy to a real server (Ubuntu). The PRD calls for an Ubuntu server
  with Nginx as a reverse proxy.

  STEPS:
  -------

  STEP 1: Get a server
  - Rent a cloud server (AWS EC2, DigitalOcean Droplet, or Hetzner)
  - Choose Ubuntu 22.04
  - Choose at least 8 cores and 16GB RAM (per PRD Section 18)
  - Get the server's IP address

  STEP 2: Set up the server
  SSH into the server: ssh ubuntu@<server-ip>

  Run these commands:
    sudo apt update && sudo apt upgrade -y
    sudo apt install -y docker.io docker-compose nginx certbot python3-certbot-nginx git
    sudo usermod -aG docker $USER
    newgrp docker

  STEP 3: Copy your code to the server
  Option A (recommended for learning): use Git
    git clone <your-repo-url> /home/ubuntu/prototype
    cd /home/ubuntu/prototype

  Option B: use scp to copy files

  STEP 4: Set up environment variables on the server
    Create /home/ubuntu/prototype/backend/.env with production values.
    Create /home/ubuntu/prototype/frontend/.env.production

  STEP 5: Build and start the application
    cd /home/ubuntu/prototype
    docker compose -f docker-compose.yml up -d --build

    Check it is running: docker compose ps
    Check logs if something fails: docker compose logs backend

  STEP 6: Configure Nginx as a reverse proxy
  Create: /etc/nginx/sites-available/financial-assistant

    server {
        listen 80;
        server_name yourdomain.com;

        # Route /api/* to FastAPI backend
        location /api/ {
            proxy_pass http://localhost:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # Route everything else to Next.js frontend
        location / {
            proxy_pass http://localhost:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }

  Enable it:
    sudo ln -s /etc/nginx/sites-available/financial-assistant \
               /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl restart nginx

  STEP 7: Enable HTTPS (free SSL certificate)
    sudo certbot --nginx -d yourdomain.com
    Follow the prompts. It auto-configures Nginx for HTTPS.
    Test: https://yourdomain.com → should load the app.

  STEP 8: Make the app restart automatically if the server reboots
    Create: /etc/systemd/system/financial-assistant.service

      [Unit]
      Description=Financial Analysis Assistant
      After=docker.service
      Requires=docker.service

      [Service]
      WorkingDirectory=/home/ubuntu/prototype
      ExecStart=/usr/bin/docker compose up
      ExecStop=/usr/bin/docker compose down
      Restart=always

      [Install]
      WantedBy=multi-user.target

    sudo systemctl enable financial-assistant
    sudo systemctl start financial-assistant

  CHECK: Reboot the server (sudo reboot), wait 2 minutes, visit
  https://yourdomain.com → app should load automatically.
